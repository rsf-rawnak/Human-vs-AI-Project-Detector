# AI Project Detector 🤖🔍

A machine learning project that detects whether a GitHub repository or a piece of code was **written by a human** or **generated by AI** — based on behavioral and structural patterns extracted from the code and commit history.

---

## What It Does

This tool analyzes GitHub projects and raw code snippets to classify them as either **Human Made** or **AI Generated**. It does this by extracting features like commit behavior, naming conventions, comment density, and AI-specific vocabulary patterns — then running them through trained ML models to give a verdict with a confidence score.

---

## Features

- Accepts a **GitHub repository URL** or a **pasted code snippet**
- Extracts 10 behavioral and structural features automatically
- Classifies using **Random Forest** and **XGBoost** models
- Shows confidence scores with a visual bar breakdown
- Deployed with a clean **Gradio web interface**

---

## Dataset

**File:** `AI_vs_Human_Dataset.csv`

- 600 samples — 300 human-written, 300 AI-generated (perfectly balanced)
- 10 features + 1 label column

| Feature | Description |
|---|---|
| `commit_count` | Total number of commits in the repo |
| `avg_commit_message_length` | Average length of commit messages |
| `commit_message_generic_score` | How generic the commit messages are (e.g. "initial commit", "update") |
| `code_comment_density` | Ratio of comment lines to total lines |
| `naming_convention_consistency` | How consistently snake_case or camelCase is used |
| `readme_complexity_score` | Complexity/length of the README file |
| `ai_vocabulary_score` | Frequency of AI-typical words in comments (e.g. "utilize", "leverage", "robust") |
| `code_structure_uniformity` | Consistency of indentation and structure |
| `large_initial_commit` | Whether the first commit added a large number of files |
| `files_added_in_one_commit` | Number of files added in a single commit |
| `label` | `0` = Human, `1` = AI Generated |

---

## How It Works

**1. Feature Extraction**

For a **GitHub URL**, the tool fetches the repo using the GitHub API and analyzes commit history, README, and source files.

For a **code paste**, features are extracted directly from the text using regex and heuristics.

**2. Model Training**

Two models are trained and compared:
- `RandomForestClassifier` — 200 estimators, max depth 10
- `XGBClassifier` — 200 estimators, learning rate 0.1

Both are trained on scaled features using `StandardScaler`.

**3. Prediction**

The trained Random Forest model is used for final predictions. Output includes:
- Extracted feature values
- Human vs AI confidence percentages (with a visual bar)
- A final verdict: `✅ HUMAN MADE` or `🚨 AI GENERATED`

## Setup & Usage

**Install dependencies:**
```bash
pip install pandas numpy matplotlib seaborn scikit-learn xgboost gradio PyGithub joblib
```

**Run the notebook:**

Open `AI_Project_Detector.ipynb` in Jupyter or Google Colab and run all cells in order:

| Cell | What it does |
|---|---|
| 1 | Installs packages and imports libraries |
| 2 | Loads and explores the dataset |
| 3 | Runs EDA with 6 visualizations |
| 4 | Preprocesses data and splits into train/test |
| 5 | Trains Random Forest and XGBoost models |
| 6 | Evaluates and compares both models |
| 7 | Shows feature importance for both models |
| 8 | Saves models and scaler to disk |
| 9 | Launches the Gradio web interface |

**Launch the web app:**

After running all cells, Cell 9 will launch the Gradio interface. A public share link will be printed automatically (`share=True`).

---

## Gradio Interface

The app has two tabs:

**GitHub URL Tab**
- Paste any public GitHub repo URL
- Click "Analyze Repository"
- The tool fetches real data from the GitHub API and runs the prediction

**Code Paste Tab**
- Paste any Python code snippet directly
- Click "Analyze Code"
- Features are extracted locally without needing a GitHub token

## Notes

- GitHub URL analysis uses the **unauthenticated GitHub API** (60 requests/hour limit). For heavier usage, add a personal access token inside the `Github()` call.
- When using the Code Paste tab, commit-related features are set to defaults since there's no git history available.
- The Gradio interface uses `share=True` — a public URL is generated automatically for sharing.
